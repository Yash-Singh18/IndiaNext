│ Implementation Phases                                                                                                        │
│                                                                                                                              │
│ Phase 1: Backend Service Skeleton                                                                                            │
│                                                                                                                              │
│ Files: phishingdetection/ (new folder)                                                                                       │
│                                                                                                                              │
│ 1. Create FastAPI app with CORS, health route on port 8001                                                                   │
│ 2. Pydantic settings loading .env (Groq keys, Supabase, VirusTotal, etc.)                                                    │
│ 3. Pydantic request/response schemas                                                                                         │
│ 4. Add to docker-compose.yml as new service (depends on qdrant, redis)                                                       │
│                                                                                                                              │
│ Phase 2: Email Parsing & Feature Extraction                                                                                  │
│                                                                                                                              │
│ Files: phishingdetection/services/email_parser.py, feature_extractor.py                                                      │
│                                                                                                                              │
│ 1. Email parser: Accept raw text paste or .eml file upload                                                                   │
│   - Strip HTML/CSS/scripts (using BeautifulSoup)                                                                             │
│   - Extract: sender, subject, body, all URLs, headers (SPF, DKIM, Return-Path)                                               │
│   - Parse .eml files (using Python email stdlib)                                                                             │
│ 2. Feature extractor:                                                                                                        │
│   - Typosquatting detection (Levenshtein distance against known brands)                                                      │
│   - Domain age check (python-whois)                                                                                          │
│   - TLD analysis                                                                                                             │
│   - Language signals (urgency keywords, financial/credential requests)                                                       │
│                                                                                                                              │
│ Phase 3: LLM Pipeline (Two-Tier)                                                                                             │
│                                                                                                                              │
│ Files: phishingdetection/services/llm_triage.py, llm_deep_analysis.py                                                        │
│                                                                                                                              │
│ 1. Triage (Llama 8B via Groq - IndiaNext1 key):                                                                              │
│   - Fast classification: SAFE / SUSPICIOUS / PHISHING                                                                        │
│   - Confidence score + brief reason                                                                                          │
│   - Route: SAFE → store directly, SUSPICIOUS/PHISHING → deep analysis                                                        │
│ 2. Deep Analysis (Llama 70B via Groq - IndiaNext2 key):                                                                      │
│   - Detailed reasoning with JSON output                                                                                      │
│   - Risk score (0-100), reasons array, suspicious URLs, recommended action                                                   │
│   - Only triggered for non-SAFE emails (saves compute)                                                                       │
│                                                                                                                              │
│ Phase 4: URL Scanner & Threat Scoring                                                                                        │
│                                                                                                                              │
│ Files: phishingdetection/services/url_scanner.py, threat_scorer.py                                                           │
│                                                                                                                              │
│ 1. URL scanner:                                                                                                              │
│   - Resolve redirects (follow chain, detect suspicious redirects)                                                            │
│   - VirusTotal API lookup                                                                                                    │
│   - Google Safe Browsing API lookup                                                                                          │
│   - Homoglyph/IDN attack detection                                                                                           │
│ 2. Threat scorer (weighted combination):                                                                                     │
│   - LLM output: 40%                                                                                                          │
│   - URL reputation: 25%                                                                                                      │
│   - Domain analysis: 20%                                                                                                     │
│   - Language signals: 15%                                                                                                    │
│   - Final score: 0-30 safe, 30-60 suspicious, 60-100 phishing                                                                │
│                                                                                                                              │
│ Phase 5: Database & API Routes                                                                                               │
│                                                                                                                              │
│ Files: phishingdetection/routes/scan.py, results.py, services/db_service.py                                                  │
│ New Supabase migration: supabase/migrations/..._create_email_scans.sql                                                       │
│                                                                                                                              │
│ 1. Supabase tables:                                                                                                          │
│ email_scans (                                                                                                                │
│   id UUID PK,                                                                                                                │
│   user_id UUID FK auth.users,                                                                                                │
│   sender TEXT,                                                                                                               │
│   subject TEXT,                                                                                                              │
│   body_preview TEXT,                                                                                                         │
│   classification TEXT,  -- safe/suspicious/phishing                                                                          │
│   risk_score INT,                                                                                                            │
│   created_at TIMESTAMPTZ                                                                                                     │
│ )                                                                                                                            │
│                                                                                                                              │
│ scan_results (                                                                                                               │
│   id UUID PK,                                                                                                                │
│   scan_id UUID FK email_scans,                                                                                               │
│   triage_output JSONB,                                                                                                       │
│   deep_analysis JSONB,                                                                                                       │
│   url_reports JSONB,                                                                                                         │
│   domain_analysis JSONB,                                                                                                     │
│   threat_breakdown JSONB,                                                                                                    │
│   reasons TEXT[],                                                                                                            │
│   recommended_action TEXT                                                                                                    │
│ )                                                                                                                            │
│   - RLS: users can only read their own scans                                                                                 │
│ 2. API routes:                                                                                                               │
│   - POST /api/scan/email — accept JSON {sender, subject, body, user_id}                                                      │
│   - POST /api/scan/eml — accept .eml file upload + user_id                                                                   │
│   - GET /api/results/{scan_id} — full scan result                                                                            │
│   - GET /api/results/user/{user_id} — all scans for a user (dashboard list)                                                  │
│                                                                                                                              │

│   - Scan results: Classification badge, risk score gauge, threat breakdown                                                   │

│   - Detail view: Full AI reasoning, URL analysis, recommended actions                                                        │
